# Code Agent 현재 구현 설명서

작성 기준일: 2026-03-12  
대상: 일반 사용자(비개발자)

## 0. 이 문서에서 다루는 범위
이 문서는 **현재 코드에 실제로 구현된 화면과 기능**만 설명합니다.

- 핵심: `설계 Kanban`, `개발 Kanban`
- 함께 설명: 프로젝트 생성/선택, 하단 로그 패널
- 참고: `대시보드`, `콘솔`, `Git 관리`, `프로젝트 설정`은 일부가 준비 중이거나 모의(샘플) 데이터 기반입니다.

---

## 1. 소프트웨어 아키텍처

### 1.1 전체 구조(쉽게 보기)

```text
[사용자]
   ↓
[Electron 데스크톱 앱 창]
   ↓
[React 화면(프론트엔드)]
   ↓ REST API
[FastAPI 서버(백엔드)]
   ↓
[MySQL DB]

(실시간 알림)
[백엔드 Agent 실행 결과] → [WebSocket] → [화면 상태 즉시 반영]
```

### 1.2 역할 분리

- Electron
  - 데스크톱 앱 창을 띄웁니다.
  - 폴더 선택 창 열기, VSCode 열기 같은 OS 기능을 연결합니다.
- 프론트엔드(React + Zustand)
  - 사용자가 보는 화면 전체를 담당합니다.
  - 프로젝트/Task/Spec 상태를 메모리에 관리합니다.
- 백엔드(FastAPI)
  - 프로젝트/Task/Spec 저장, 수정, 삭제 API를 제공합니다.
  - 설계 Agent/개발 Agent를 백그라운드로 실행합니다.
- DB(MySQL)
  - 프로젝트, 스펙, 태스크, 상태값을 저장합니다.
- WebSocket
  - AI 분석 진행, Task 상태 변경(coding/reviewing/done 등)을 실시간으로 전달합니다.

### 1.3 현재 구현에서 중요한 포인트

- 화면의 대부분은 `http://localhost:8000` API를 호출해 데이터 처리합니다.
- 설계/개발 단계는 WebSocket(`ws://localhost:8000/ws/{project_id}`)으로 상태를 실시간 반영합니다.
- 에이전트 실행은 백엔드에서 비동기(백그라운드)로 처리됩니다.

## 2. 현재 화면과 기능

### 2.1 공통 화면 구성

- 상단 헤더
  - 앱 이름, 현재 사용자 ID 표시
  - 알림/설정 아이콘(현재는 시각 요소 중심)
- 좌측 사이드바
  - 프로젝트 목록
  - 메뉴 이동: 설계 Kanban, 개발 Kanban, 콘솔, Git 관리, 프로젝트 설정
  - 새 프로젝트 생성 모달
- 하단 패널
  - 탭: 로그 / 실행 결과 / Git 상태
  - 현재는 샘플 로그 중심 표시

### 2.2 프로젝트 생성/선택

- 프로젝트 생성 가능 항목
  - 기존 프로젝트 / 신규 프로젝트 선택
  - 프로젝트 이름
  - 로컬 경로(필수, 폴더 선택 버튼 지원)
  - Git URL(선택)
  - 메인 브랜치, 설명
- 생성 성공 시
  - 프로젝트 목록에 즉시 추가
  - 새 프로젝트가 선택됨

### 2.3 설계 Kanban 화면

- 화면 목적
  - 요구사항(Spec)을 업로드하고, AI로 Task로 분해한 뒤 확정
- 주요 기능
  - Spec 추가: 파일/이미지/텍스트 입력/드래그 앤 드롭
  - 분석 시작 버튼
  - Task 편집(제목/설명/우선순위/복잡도/수용 기준)
  - Task 삭제
  - 드래그 앤 드롭으로 상태 이동
  - `전체 확정` 버튼
  - `Spec 확정` 버튼

### 2.4 개발 Kanban 화면

- 화면 목적
  - 확정된 Task를 AI 개발 사이클로 실행하고 완료 상태까지 추적
- 주요 기능
  - 4개 컬럼: Backlog / In Progress / AI Review / Done
  - 카드 드래그 이동
  - Backlog 카드의 `실행` 버튼으로 에이전트 실행
  - 프로젝트 로컬 경로가 있으면 `VSCode 열기`

### 2.5 그 외 화면(현재 상태)

- Dashboard: `개발 예정` 표시
- Console: 명령 입력 UI는 있으나 실제 백엔드 실행 연동은 아직 준비 단계(샘플 로그 기반)
- Git 관리: 화면 UI는 있으나 현재 샘플 데이터 중심 동작
- 프로젝트 설정: 읽기 전용 예시 값 표시(상세 설정은 추후)

## 3. 설계 KANBAN 작동 흐름

아래는 사용자가 실제로 보는 흐름입니다.

1. 프로젝트 선택
- 왼쪽 프로젝트 목록에서 작업할 프로젝트를 선택합니다.

2. Spec 입력
- 파일 업로드, 이미지 업로드, 텍스트 입력, 드래그 앤 드롭 중 하나로 Spec을 넣습니다.
- 입력된 Spec은 `Spec 입력` 컬럼에 나타납니다.
- 참고: 현재 서버 기준 문서 업로드 허용 형식은 `pdf/doc/docx`입니다.

3. 분석 시작
- Spec 카드의 `분석 시작` 버튼을 누르면 분석이 시작됩니다.
- Spec은 `AI 분석중` 컬럼으로 이동해 진행 상태를 보여줍니다.

4. AI가 Task 생성
- 백엔드 설계 Agent가 Spec을 읽고 Task 목록으로 분해합니다.
- 완료되면 Task 카드가 `Task 분해됨` 컬럼에 생성됩니다.

5. Task 검토/수정
- 카드를 클릭해 상세 수정(제목, 설명, 수용 기준 등) 가능
- 필요 없는 Task는 삭제 가능

6. 확정 처리
- 방법 A: 드래그로 `Task 분해됨` → `확정됨`
- 방법 B: `전체 확정` 버튼으로 일괄 이동

7. Spec 최종 확정
- `확정됨` 컬럼에서 `Spec 확정` 버튼을 누르면 해당 Spec이 최종 확정됩니다.

### 설계 단계 상태 변화(내부)

- Spec: `uploaded → analyzing → (분석 완료 후 목록 재반영) → final_confirmed`
- Task: `plan_reviewing → confirmed`

---

## 4. 개발 KANBAN 작동 흐름

1. 시작점
- 설계 단계에서 `confirmed`가 된 Task가 개발 Kanban의 Backlog에 보입니다.

2. 실행 또는 수동 이동
- Backlog 카드의 `실행` 버튼으로 AI 개발 시작
- 또는 드래그 앤 드롭으로 상태를 수동 변경 가능

3. AI 개발 사이클(백엔드 오케스트레이터)
- `confirmed → coding → reviewing`
- 리뷰 통과 시 `done`
- 리뷰 실패 시 피드백을 반영해 재시도(최대 3회)
- 재시도 초과/오류 시 `failed`

4. 실시간 반영
- 상태 변경은 WebSocket으로 화면에 즉시 반영됩니다.

### 개발 단계 상태 변화(내부)

- 기본 표시 컬럼: `confirmed`, `coding`, `reviewing`, `done`
- 실패 상태 `failed`는 `done`에 같이 표시 됩니다.

## 5. 현재 구현 범위 요약

- 실제 핵심 구현
  - 프로젝트 생성/선택

  - 개발 Kanban: 실행/상태 이동/실시간 상태 반영
- 부분 구현(준비 단계)
  - Dashboard
  - Console 실연동
  - Git 관리 실연동
  - 설정 상세 편집
  - 문서 업로드 형식 일부(UI 안내와 서버 허용 범위) 정리 필요
